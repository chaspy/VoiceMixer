# VoiceMixer 仕様確定メモ / 設計 / 実装方針

最終更新日: 2026-02-21
対象: ローカル実行ブラウザアプリ VoiceMixer（PC Chrome/Edge想定）

## 1. 目的
口で多重録りして作曲/編曲ラフを作り、最終的に1本の音源へミックスダウンして手動で外部サービスへ投入できる状態にする。

## 2. 前提・制約
- ローカル実行（オフライン可）
- TypeScript 実装
- ブラウザ動作（まずは PC Chrome/Edge）
- 外部サービス連携なし（Suno API 等は不要）

## 3. ユーザ合意済み仕様（確定）
### ワークフロー
- 通し録り専用ではなく、1曲タイムライン上で以下を混在可能にする
  - 部分録音
  - 最初から録音
  - 途中から録音（パンチイン）

### テンポ/拍子
- 固定テンポ（BPMあり）
- 拍子は 4/4 を初期値とする

### トラック
- 通常想定は 8 トラック
- 最大 16 トラックまで対応

### 編集機能（MVP）
- トリム（前後）
- パンチイン
- クリップの位置移動（細かい調整可能）
- 移動時は「常に自由移動（スナップなし）」
- ループ範囲指定: MVP対象外
- フェード: MVP対象外

### パンチイン時の既存クリップ処理
- 非破壊分割（既存クリップを前後に分割し、対象区間のみ差し替え）

### ピッチ処理
- 可視化あり
- 簡易ピッチ補正あり
- 補正方式は「オフライン補正（録音後適用）」

### 口ドラム
- MVPに含める
- 3クラス置換（Kick/Snare/Hi-hat）
- 判定方式は閾値ベース簡易分類（オンセット + 帯域/特徴量）

### 取り込み
- MVPでは「録音のみ」
- 既存ファイル（WAV/MP3）取り込みは次フェーズ

### 出力
- ユーザ意図は MP3 だったが、MVP方針として MP3 は次フェーズ
- MVPは WAV 書き出しを提供（ノーマライズ任意）

### 完成ライン
- MVPでまず利用開始し、少人数評価後に改善

### 最大長
- 1プロジェクト最大長は 10分

## 4. MVP機能要件
1. プロジェクト作成/保存
- BPM・拍子・プロジェクト長を設定可能
- IndexedDBへ保存

2. トラック管理
- 追加/削除（上限16）
- 名前、ミュート、ソロ、音量、パン、録音アーム

3. 録音
- カウントイン後に録音開始
- 通常録音
- パンチイン録音（非破壊分割）

4. タイムライン編集
- トリム
- 自由移動（ドラッグ、微調整）

5. メトロノーム
- BPM同期クリック（ON/OFF、音量）

6. ピッチ
- 選択クリップのピッチ可視化
- オフライン簡易ピッチ補正

7. 口ドラム置換
- 3クラス推定して置換トラック生成

8. 書き出し
- OfflineAudioContext でミックスダウン
- WAV書き出し

## 5. 採用技術スタック
- フロント: Vite + React + TypeScript（strict）
- 音声: Web Audio API
- ピッチ解析: pitchy
- 保存: IndexedDB（idb）
- 書き出し: OfflineAudioContext + WAVエンコード

## 6. モジュール設計（確定）
- Transport: 再生位置・再生/停止/シーク
- Recorder: MediaRecorder 制御
- Timeline: クリップ配置、トリム、パンチイン適用
- Mixer: mute/solo/volume/pan
- Pitch: 解析と簡易補正
- DrumReplace: 口ドラム検出・置換
- Renderer: OfflineAudioContext ミックスダウン
- Storage: IndexedDB 永続化
- UI: Project一覧、Mixer/Timeline、Pitchビュー、Export

## 7. データモデル（要点）
- Project: bpm, timeSig, lengthSec, tracks
- Track: mute, solo, volume, pan, armed, clips
- Clip: blob, startSec, offsetSec, durationSec, sourceDurationSec, pitch
- DrumHit: timeSec, type(kick/snare/hihat), strength

## 8. エラーハンドリング方針
- マイク拒否: 復帰可能なUIメッセージ
- デコード失敗: 対象クリップ単位で失敗通知
- 長尺/高負荷: 10分制限で事前回避
- 録音中断/再生競合: 状態遷移を明確化

## 9. 非MVP（次フェーズ）
- MP3書き出し
- WAV/MP3インポート
- ループ範囲指定
- フェード
- Undo/Redo強化
- 量子化
- ドラム変換精度向上（ML等）

## 10. 受け入れ基準（MVP）
- 録音→編集（トリム/移動/パンチイン）→再生→WAV書き出しがローカル完結で動く
- 最大16トラックまで操作できる
- ピッチ可視化と簡易補正が実行できる
- 口ドラム置換でドラムトラックを生成できる
